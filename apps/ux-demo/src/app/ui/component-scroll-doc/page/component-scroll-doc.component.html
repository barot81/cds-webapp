Component Scroll Layout

Problem Statement
In current layouts, if we set the page level scroll at the parent level then whichever page will render in the router-outlet will have page level scroll, due to this if we need to add component level scroll in child layout, it will cause a problem to have multiple scroll handler and also, we cannot achieve the sticky header and left sidebar layout.
Solution
Will not have page level scroll on any page. Will only have component level scroll using header service.

Additional points

  • Body area of any grid vary between screen to screen. In some screen you see more record whereas in some screen you might relatively less.
  • No avatar on grid because we are using condensed version of grid to see more record compare to regular grid.
  • Whenever we have list of people only student module will have avatar.
  • We will have component scroll for right sidebar irrespective of it being a card, table, accordion etc , in order to keep left sidebar fixed.
  • We can give sticky to table footer, in order to make it easily accessible to users. In simple words, table body will scroll with respect to table head and table footer.
  • In search bar we are showing search icon. When user search, we want to introduce “X” / cancel icon to remove the search query.
  • One cell one value – condense grid.
  • Same screen will appear slightly different appearance based on the machine it gets open.
